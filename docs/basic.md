# 基本素养

## 信息素养

什么是信息素养？

信息素养是信息化时代个体**获取、评估与利用信息的综合能力**。1989 年，美国图书馆协会（ALA）给出了一个被广泛引用的经典定义："要成为一个有信息素养的人，就必须能够确定何时需要信息，并且能够有效地查寻、评价和使用所需要的信息。"

对刚进入大学、开始学习计算机的同学来说，信息素养至少包括：

- 信息获取能力：通过搜索引擎、数据库、图书馆等多种途径，快速准确地找到所需信息。
- 信息处理能力：对收集到的信息进行整理、分类、分析和综合，提取有效内容。
- 信息评价能力：判断信息的来源可靠性、真实性和价值，辨别信息真伪，避免被误导。
- 信息利用能力：将信息应用于解决实际问题，进行创新和决策。
- 信息传播能力：以适当的方式分享和传播信息，同时遵守版权、隐私等相关法规。

具体而言：

<div class="grid cards" markdown>

- **明确问题**

    先把“我不会”“它报错了”拆成更具体的问题，例如“我在 Windows 11 上用 VS Code 编译 C 语言，终端提示 `gcc` 不是内部或外部命令”。

- **选择来源**

    软件下载优先找官网或可信的应用商店；编程问题优先看官方文档、权威手册、项目仓库的 README/Issue，再看博客、问答和视频。

- **判断可信度**

    看发布者是谁、发布时间是否过旧、内容是否对应你的系统/版本、是否有可复现的步骤、多个来源是否互相印证。

- **警惕诱导**

    搜索结果的第一条不一定最好，带“广告”“高速下载”“安全下载”“破解版”“绿色版”的页面尤其要小心。

- **整理和复盘**

    解决问题后记下关键词、链接、命令、环境和最终原因。下次遇到类似问题，你会更快定位。

- **负责使用信息**

    不要随手复制来路不明的命令；不要把账号、Cookie、密钥、身份证号、学生信息贴给 AI 或发到公开群里；引用他人内容时保留来源。

</div>

简单说，信息素养的目标是让你少被垃圾信息牵着走，少在重复问题上浪费时间，并且能独立完成“发现问题 -> 查找资料 -> 判断资料 -> 解决问题”的闭环。

一个不掌握信息素养的可预见后果是，你想下载 steam 玩游戏，打开**百度**搜索“steam”并点进了第一条结果，在广告的诱导下下载了“steam 管家”（虽然现在貌似有所改进）。

## 提问交流

凌睿工作室欢迎同学们积极提问和讨论，当你无法通过自助手段解决问题时，向他人求助是一个好选择。使用合适的方式提问可以节省各方的时间并且提高解决问题的可能性。以下是有效提问的关键要素：

- 没有笨的问题：任何问题都有被提出的价值，把问题憋在心里只是阻碍自己的进步。
- 描述清晰：清楚地描述你的问题背景，你尝试过的解决方法和你当前的困惑。问题要具体，避免笼统或含糊不清的描述。
- 选择合适的被提问方：在多数情况下，他人没有义务或能力解答你。
- 尊重他人时间：理解他人可能有自己的工作安排，礼貌提问，耐心等待回复。

若是与工作室/招新相关，很多问题在本站顶栏搜索框中搜索就能找到答案；没有的话先看看招新群公告、文件、历史记录。

配图时请**不要用手机拍屏幕**！请你在电脑上截图！若是要发到 QQ，QQ 聊天框自带截图按钮；若是在其他地方截图，推荐使用 [PixPin](https://pixpin.cn/)。

## 善用搜索引擎

- 不要使用**百度**！优先推荐 Bing（Windows 电脑的 Edge 浏览器默认搜索引擎），最方便；能用 Google、DuckDuckGo 也很好。
- 搜索时不要输入一整句口语。把问题拆成关键词：对象、现象、环境、版本、报错信息。例如不要只搜“为什么运行不了”，而是搜 `VS Code C gcc 不是内部或外部命令 Windows 11`。
- 技术问题尽量用英文关键词。很多编程资料、官方文档和报错讨论都是英文的，搜索 `python ModuleNotFoundError requests` 往往比搜“python 找不到 requests 模块”更容易找到有效答案。
- 报错信息要原样复制。把核心报错用英文双引号包起来可以精确匹配，例如 `"No such file or directory"`、`"Permission denied"`。如果路径、用户名、学号等包含隐私，先删掉或替换。
- 加上限定条件。系统、软件版本、编译器、框架版本都可能影响答案，例如 `Ubuntu 24.04 gcc install`、`Python 3.12 pip install package`、`Vue 3 vite alias`。
- 优先寻找一手资料。下载软件找官网；查 Web API 看 MDN；查 C/C++ 标准库看 cppreference；查 Linux 问题可参考 Arch Wiki、Ubuntu 文档、项目官方文档；查课程、通知和政策以学校/学院官网为准。
- 不要只看第一条结果。先快速扫标题、网址、摘要和时间，避开广告、搬运站、内容农场、明显过时的教程，再点进去看。
- **询问 AI**。AI 太好用了你知道吗！DeepSeek、千问、Kimi、智谱、ChatGPT 等大模型很适合解释概念、整理思路、生成搜索关键词和辅助排错。但它给的不一定准确和完善，必要时仍然需要自行搜索。
