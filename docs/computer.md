# 电脑小白扫盲

在此之前，可能有同学从未真正拥有过一台自己的电脑，或是用过但了解不深。因此，一个扫盲教程还是很有必要的。

这篇文章不打算把你变成硬件发烧友，也不打算教你把系统折腾到多么花里胡哨。它的目标很朴素：让你知道电脑是什么、该怎么买、买来之后该怎么用，以及少踩一些非常低级但非常浪费时间的坑。

如果你已经有电脑了，也不必急着换。先看完这篇，判断它能不能满足学习和开发需求；如果只是配置环境、写代码、做课程实验，大多数近几年的电脑都还可以继续用。

## 操作系统

操作系统是一台电脑的灵魂，你打开的所有程序都运行在操作系统之上。

消费级电脑最常见的操作系统主要是 Windows 与 macOS，它们对普通用户比较友好。但在程序员圈子中，还有一个非常常用的操作系统——Linux。

把它们放在一起对比，有几个我们需要考虑的维度：

- 兼容性：Windows > macOS > Linux。几乎所有游戏、网课软件、办公软件、专业软件都能在 Windows 上打开，它甚至兼容很多二十多年前的文件格式。macOS 的常用软件也很完善，且近几年开发者生态越来越好，但某些课程软件、考试软件、行业软件仍然只支持 Windows。至于 Linux，也只有程序员可能把它当作日常主力了。
- 开发便捷性：Linux > macOS > Windows。Windows 虽然有着最强的兼容性，但随之而来的是无数历史遗留问题和无法改进的设计缺陷，这导致基于 Windows 环境的开发往往会遇到各种问题，参考 [停止用 Windows 工作！- 知乎](https://zhuanlan.zhihu.com/p/2024527609388627701)。而 Linux 和 macOS 都是 Unix-like OS，许多事情一两行命令即可搞定。然而，微软似乎也知道 Windows 开发有多难受，于是现在的 Windows 都支持了 WSL2（Windows Subsystem for Linux），它是在 Windows 中运行 Linux 环境的官方方案，算是 Windows 在编程领域的杀手锏了。
- 折腾成本：Windows 和 macOS 都相对低，Linux 取决于你怎么用。Linux 作为服务器和开发环境很好，但作为第一台电脑的唯一桌面系统，我不建议小白一上来就这么干。

简单说：

- 如果你不知道选什么，选 Windows。
- 如果你已经有 MacBook，不用因为学计算机就换掉它。
- 如果你学编程，那迟早都要接触 Linux，但不一定要把电脑装成 Linux。其实 Windows WSL2 就很好用。

顺带一提，Windows 10 已于 2025 年 10 月 14 日结束常规支持，新电脑不要再买 Windows 10 了。除非你明确知道自己在干什么，否则直接用 Windows 11。

## 电脑选购

不知道看到这篇文档的你是否已经买了电脑，如果没有，那你可得好好看看。

市面上的电脑一般分为两种：笔记本电脑和台式电脑。这里我们主要讨论笔记本电脑，也推荐电脑小白买笔记本电脑作为其第一台电脑。

原因很简单：大学生活不是固定坐在一个工位上，你会把电脑带去教室、图书馆、实验室、答辩现场、组会现场，甚至只是从宿舍桌子挪到床上。并且笔记本作为一个整体，有厂商调教过，不需要怎么折腾。

### 笔记本类型

笔记本一般分为轻薄本、全能本、游戏本。

- 轻薄本：无独显、重量轻、续航好、噪音小，适合上课、写代码、写文档、做轻量实验。缺点是显卡性能通常一般，不适合重度游戏、3D 渲染。
- 全能本：介于轻薄本和游戏本之间。一般来讲是集两家之短，又不便携，噪声又大，不推荐。
- 游戏本：带独显、性能强、散热堆料足，适合游戏、本地渲染、剪视频。缺点也很明显：重、续航差，充电器像板砖。你愿意每天背着它上课吗？想清楚。

对于大多数计算机相关专业新生，轻薄本就够了。现在很多游戏已经不需要独立显卡也能带得动了，除非你就必须要拉满画质和帧率打游戏、或是本地跑 ai 训练。

### 选购时真正重要的配置

选电脑不要只看商家大字写的“i7”“高性能”“电竞级”“AI PC”。那些词往往没什么信息量。你应该重点看下面这些。

#### 内存

内存就是 RAM，不是硬盘空间。它决定了你同时开多少软件、浏览器开多少标签、虚拟机能不能流畅跑。

- 8GB：不推荐。能用，但很快就会让你痛苦。
- 16GB：目前的最低建议配置，适合预算有限、需求普通的同学。
- 32GB：更推荐。写代码、开浏览器、跑 Docker、开虚拟机、做课程项目都会舒服很多。
- 64GB 及以上：除非你明确知道自己需要，否则没必要。

需要注意的是，很多轻薄本内存是板载的，后期不能升级。所以买之前别想着“以后再加”，你很可能根本加不了。

#### 硬盘

硬盘就是存储空间，决定了你能放多少软件、项目、视频、游戏、虚拟机镜像。

- 512GB 及以下：不推荐，除非你非常确定自己只做轻量使用。
- 1TB：更推荐。装系统、开发工具、课程资料、几个虚拟机、几个游戏之后，你会感谢当初多买了这点空间。
- 2TB 及以上：适合一劳永逸。

硬盘类型要选固态硬盘（SSD），现在新电脑基本都是 SSD，留意一下即可。

#### CPU

CPU 是电脑的大脑，但不要迷信“i9 一定比 i7 强”“数字越大越好”。笔记本 CPU 还要看功耗、散热、代际、核心设计。

对于新生来说，比较稳妥的原则是：

- 买近两三年的主流平台，不要买库存老古董。
- 不要为了 i9 多花很多钱，很多轻薄本上的 i9 因为散热压不住，实际体验未必比 i7 好。
- Intel、AMD、Apple Silicon 都能用，重点看整机评测，而不是只看芯片名。
- 看到“赛扬”“奔腾”“N 系列低功耗处理器”等超低端平台时要谨慎，它们可能适合办公看网课，但不适合作为计算机学生的主力机。

硬件每年都变，别把某一代芯片当成永恒答案。选购时看当年的靠谱评测，比记几个型号名重要得多。

#### 显卡

显卡主要影响游戏、3D、视频剪辑、深度学习训练、CUDA 相关任务。

- 不玩大型游戏、不做本地 AI 训练：核显足够。
- 玩 3A 游戏、做渲染、用 CUDA：优先考虑 NVIDIA 独显。
- 想学深度学习：很多任务可以用云平台、实验室服务器或远程机器，不一定非要本地跑。

如果你实在有高级显卡的需求，我更推荐搭配台式电脑或远程服务器，日常携带轻薄本用于远程控制。这样体验通常比天天背游戏本好。

#### 屏幕

屏幕是你每天盯着看的东西，别太省。

建议关注：

- 尺寸：14 英寸便携，16 英寸舒服。13 英寸以下对写代码来说偏小。
- 分辨率：1080p 能用，2.5K/2.8K 更舒服。4K 在笔记本上未必有必要，还会增加耗电。
- 亮度：300 nit 是基本能用，400 nit 及以上户外和明亮环境更舒服。
- 色域：写代码要求不高，但如果你做设计、视频、摄影，至少要关注 sRGB 覆盖。
- 刷新率：高刷对日常流畅感有帮助，但不是刚需。

不要被“电竞屏”“护眼屏”“原色屏”这种宣传词直接说服，尽量看具体参数和评测。

#### 续航与充电

续航是笔记本体验的核心之一。商家标称的续航通常是在非常理想的条件下测出来的，实际使用会短很多。

建议：

- 看独立评测的真实办公续航，而不是只看宣传页。
- 优先选择支持 USB-C PD 充电的机器，出门带一个小充电器会舒服很多。
- 游戏本基本不要期待离电性能和长续航。
- 续航、性能、重量三者不可兼得，别幻想一台电脑又轻又强又便宜还续航十几个小时。

#### 接口

接口少不一定不能买，但你要知道自己需要什么。

常见接口包括：

- USB-A：接鼠标、U 盘、键盘、一些老设备。
- USB-C：充电、扩展坞、外接显示器、传数据。
- HDMI：接投影、显示器。
- 3.5mm 耳机孔：有线耳机。
- 网线口：游戏本和部分全能本常见，轻薄本较少。
- SD 卡槽：摄影、视频同学可能需要。

轻薄本接口少，可以靠扩展坞解决，但扩展坞也要钱，也会带来兼容性问题。

#### 键盘、触控板和重量

这些参数看起来不如 CPU、显卡“硬”，但直接决定你每天用电脑的心情。

- 键盘手感太差，写代码会很难受。
- 触控板太小或误触严重，移动办公会很烦。
- 机身太重，你会逐渐不想把它带出宿舍。
- 风扇太吵，图书馆和课堂里会很尴尬。

有条件的话去线下摸一摸真机，但不要在线下店被导购忽悠下单。

### 推荐配置

如果你完全不知道怎么选，可以按下面这个范围来。

- 系统：Windows 11
- 类型：轻薄本或全能本
- 内存：16GB 起步，优先 32GB
- 硬盘：512GB 起步，优先 1TB
- 显卡：核显即可

综合来看，笔者个人最推荐的是一台靠谱的 Windows 轻薄本：去哪儿都方便，有最强大的兼容性，编程在 WSL2 中完成。若你很喜欢 macOS，也能接受价格和兼容性问题，MacBook 当然也可以。

### 不推荐购买的电脑

以下几类电脑，小白尽量别碰：

- 内存只有 8GB 且不可升级的电脑。
- 硬盘只有 256GB 的电脑。
- 标着“商务办公”“学习网课”但配置很低的廉价本。
- 来路不明的二手机、矿本、翻新机。
- 线下店导购强推但网上几乎查不到型号和评测的机器。
- 为了便宜买的 Windows 10 老库存。
- 只因为外观好看、颜色好看、明星代言就买的机器。
- 你不理解优缺点的奇怪形态设备，例如双屏本、折叠屏本、超小掌机电脑。

不是这些设备一定不能用，而是它们不适合作为小白的第一台主力机。你缺少判断能力时，越“特别”的东西越容易坑你。

### 购买渠道

购买渠道上，推荐在**京东自营**购买。

不推荐小白去电脑城和线下小店买。不是所有线下店都坑，但你没有足够判断能力时，很容易被“这个更适合学生”“这个今天活动价”“这个配置更高”之类的话术带走。

买之前尽量做到：

- 确定完整型号，不要只说“某品牌 i7 笔记本”。
- 去 B 站、知乎、贴吧、酷安、什么值得买等平台查真实评价，但不要只看一篇。
- 看差评，尤其看品控、售后、屏幕、噪音、续航、发热。
- 确认是否支持无理由退货、保修几年、是否全国联保。
- 问清楚内存和硬盘是否可升级。

### 收货验机

收到电脑后，先别急着把所有软件都装上。建议按顺序做这些事：

1. 检查包装是否破损，封条是否异常。
2. 检查机身外观、屏幕、键盘、接口有没有明显损伤。
3. 对照订单确认型号、颜色、内存、硬盘、显卡是否一致。
4. 开机进入系统后，在系统信息中确认 CPU、内存、硬盘容量。
5. 检查屏幕有没有明显坏点、亮斑、色块。
6. 测试键盘、触控板、Wi-Fi、蓝牙、摄像头、扬声器、麦克风、充电。
7. 确认没有问题后，再进行长期使用和激活 Office 等操作。

有些平台对激活系统、激活 Office、拆封、联网后的退换政策比较敏感，具体以购买渠道规则为准。小白至少要记住一句：保留包装和发票，发现硬件问题尽快联系售后，不要自己拆机修。

## Windows 必知必会

小白拿到 Windows 电脑后的第一个要注意的点就是**不要用中文用户名**！

因为 Windows 许多依赖都放在 `C:\Users\<用户名>\` 目录下，同时，不是所有程序都支持路径中包含中文字符。路径中有中文时，日常使用可能没感觉，但装编程工具、跑脚本、配环境时就可能出现玄学报错。

如果你已经用了中文用户名，也不是世界末日，但不要随便手动重命名，更稳妥的做法是新建一个英文用户名的本地管理员账户，然后迁移文件。

### 文件与目录

很多电脑小白最大的问题不是不会写代码，而是连文件在哪里都不知道。

#### 文件、文件夹和路径

文件就是具体的数据，例如一张图片、一份文档、一段代码。文件夹就是用来放文件的容器。路径就是文件或文件夹在电脑里的位置。

Windows 路径通常长这样：

```text
C:\Users\Alice\Downloads\demo.zip
D:\Code\hello\main.c
```

其中 `C:`、`D:` 是盘符，反斜杠 `\` 用来分隔目录。你不必一开始就理解 Windows 文件系统的所有细节，但至少要能在文件资源管理器顶部的地址栏里看懂、复制和修改路径。

如果你经常“明明下载了，但不知道下载到哪了”，推荐安装 [Everything](https://www.voidtools.com/)。它可以非常快地搜索 Windows 本地文件，比文件资源管理器自带的搜索好用得多。不过它只能帮你找到文件，不能替你整理文件。

#### 文件扩展名

文件扩展名是文件名最后的 `.xxx`，例如：

- `main.c`
- `report.docx`
- `image.png`
- `archive.zip`
- `script.py`

扩展名通常表示文件类型。小白一定要在文件资源管理器中打开“查看 -> 显示 -> 文件扩展名”。否则你很容易把 `作业。docx.exe` 当成 Word 文档，把 `main.c.txt` 当成 C 语言源文件。

#### 压缩文件

你会经常遇到 `.zip`、`.7z`、`.rar`、`.tar.gz` 等压缩文件。Windows 自带的解压功能能处理简单的 ZIP 文件，但功能比较有限。

这里推荐安装 [7-Zip](https://www.7-zip.org/)。它免费、开源、没有广告，支持绝大多数常见压缩格式。安装后右键压缩包，就可以通过 `7-Zip` 菜单解压。

要注意：

- 收到压缩包后先解压，再打开里面的项目或文档。
- 不要直接在压缩包内部编辑代码和作业。
- 不要运行来路不明压缩包里的 `.exe`、`.bat`、`.cmd`、`.scr` 文件。
- 解压失败时先确认文件是否下载完整，不要立刻怀疑 7-Zip。

#### 不要把桌面当仓库

桌面可以临时放东西，但不应该成为你的所有文件仓库。

建议至少建立几个固定目录：

```text
D:\Code
D:\Course
D:\Documents
D:\Temp
```

如果电脑只有 C 盘，也可以放在用户目录下：

```text
C:\Users\Alice\Code
C:\Users\Alice\Course
```

代码项目建议统一放在英文路径下。不要把代码放在下面这些路径里：

- 包含中文的路径。
- 包含奇怪符号的路径。
- 很深很乱的路径。
- OneDrive 等网盘同步目录。
- QQ/微信文件自动保存目录。
- 压缩包内部。

编程工具对路径很敏感。很多报错不是你代码写错了，而是路径里有中文、空格、权限、同步盘、压缩包这些麻烦东西。

#### 下载目录不是垃圾桶

浏览器下载的东西一般会进入 `Downloads`。很多同学的下载目录里会堆满安装包、压缩包、重复文件、临时图片。

建议养成习惯：

- 安装包用完就删，除非你明确要保留。
- 压缩包解压后确认不需要就删。
- 课程资料移动到对应课程文件夹。
- 重要文件不要只放在下载目录。

如果你发现 C 盘突然满了，可以用 [WizTree](https://diskanalyzer.com/) 查看哪些目录占用了空间。它适合找大文件，不是让你看到不认识的系统文件夹就随便删。

电脑不是因为“用久了自然变卡”，很多时候是因为你真的把它用成了垃圾堆。

### 软件安装

软件安装是小白最容易中招的环节。

#### 从哪里下载软件

优先级大概是：

1. 软件官网。
2. Microsoft Store。
3. GitHub Releases 或项目官方文档给出的下载链接。
4. Windows 自带的包管理器 `winget`。

不推荐：

- 某某软件园。
- 高速下载器。
- 绿色版、破解版、精简版。
- 来路不明的网盘链接。
- 搜索结果第一条广告。

你想下载 VS Code，就去 VS Code 官网；想下载 7-Zip，就去 7-Zip 官网。不要在百度里搜一个软件名，然后点进看起来很像官网但域名完全不对的网站。

`winget` 可以在终端里搜索、安装和升级软件，例如：

```powershell
winget search 7zip
winget install 7zip.7zip
winget upgrade --all
```

命令行安装不一定比点安装包更“高级”，但它通常能减少点进假官网和捆绑下载器的机会。

#### 安装时看清楚每一步

安装软件时不要一路“下一步”。很多软件会在安装过程中默认勾选：

- 安装其他软件。
- 修改浏览器主页。
- 加入开机启动。
- 创建一堆桌面快捷方式。
- 上传使用数据或开启云同步。

看清楚再点。你多看十秒钟，可能少卸载十个软件。

#### 安装位置

软件装在 C 盘不等于电脑会坏。Windows 下很多软件默认装在 `C:\Program Files`，这是正常的。

真正应该注意的是：

- C 盘要保留足够剩余空间。
- 大型游戏、素材库、虚拟机镜像可以放到空间更大的盘。
- 不要为了“省 C 盘”把所有软件乱装到奇怪路径。
- 开发工具尽量装在默认路径，除非教程明确要求。

#### 卸载软件

删除桌面图标不等于卸载软件。桌面图标通常只是快捷方式。这里推荐使用 [Geek Uninstaller](https://geekuninstaller.com/) 来深度卸载，但也不要拿着它乱删系统组件。

此外，以下软件你可能在 geek 中搜不到：

- Steam、Epic 等平台下载的游戏
- 便携版软件
- 盗版游戏

对于这些，你要记住他们的安装路径（或用 Everything 搜索），手动清理。

#### 杀毒软件

Windows 自带 Microsoft Defender，对普通用户已经够用，小白没有必要一上来安装一堆安全卫士和电脑管家。

如果你不喜欢 Defender，或者需要更直观的弹窗拦截、启动项管理和右键扫描，可以考虑安装 [火绒安全](https://www.huorong.cn/)。它比较安静，适合不想被各种“加速球”和广告打扰的人。

Defender 和火绒选一个正常使用即可，不要再叠加安装 360 安全卫士、腾讯电脑管家等一大堆同类软件。安全软件装太多不但不能让电脑更安全，还可能互相打架、占用资源、弹广告、拦截编译器。

真正有效的安全习惯是：

- 软件从官网下载。
- 系统和浏览器保持更新。
- 不运行来路不明的 `.exe`。
- 不打开陌生压缩包里的可执行文件。
- 不相信“破解版”“激活工具”“外挂”“刷课神器”。
- 重要账号开启双重验证。

### 任务管理器

电脑卡的时候，先打开任务管理器看一眼。

重点看：

- CPU：是否有程序一直占满。
- 内存：是否已经爆了。
- 磁盘：是否长期 100%。
- 网络：是否有程序疯狂上传下载。
- 启动应用：是否有太多软件开机自启。

不要看到一个进程名字不认识就结束。系统进程乱杀可能导致蓝屏、黑屏、软件崩溃。看不懂就先搜索进程名。

### PowerShell、CMD、Windows Terminal 和 WSL

Windows 下经常看到几个“黑窗口”，它们不是一回事。

- CMD：老牌命令提示符，很多旧教程会用。
- PowerShell：更现代的 Windows 命令行。
- Windows Terminal：一个终端外壳，可以同时打开 PowerShell、CMD、WSL 等。
- WSL：Windows Subsystem for Linux，在 Windows 中运行 Linux 环境。

如果教程让你打开“终端”，优先使用 Windows Terminal。需要运行 Linux 命令时，进入 WSL；需要运行 Windows 命令时，用 PowerShell。

你可以通过点击任务栏中的 Windows 图标 - 搜索对应名称来找到它们。

### 环境变量 PATH

很多教程会提到“配置环境变量”。其中最常见的是 `PATH`。

简单说，`PATH` 告诉系统去哪些目录里找命令。比如你在终端输入：

```powershell
python
```

系统会去 `PATH` 里的目录查找有没有 `python.exe`。如果没找到，就会出现类似“不是内部或外部命令”的提示。

小白配置环境变量时要注意：

- 不要乱删已有内容。
- 添加路径时确认路径真实存在。
- 改完后重开终端。
- Windows 的环境变量和 WSL/Linux 的环境变量不是一套。

### Windows 更新和驱动

Windows 更新虽然有时烦人，但不要长期关闭。系统更新包含安全修复、驱动更新和兼容性修复。

驱动方面：

- 普通用户优先使用 Windows Update 和品牌官方管家。
- 显卡驱动去 NVIDIA、AMD、Intel 官网或官方软件更新。
- 不建议使用来路不明的驱动精灵、驱动人生之类的软件。
- BIOS 更新有风险，除非官方明确修复重要问题或你知道自己需要，否则不要随便更。

## 网络与代理

能打开 QQ、微信、B 站，不代表你的网络环境适合学习计算机。

计算机相关学习会频繁接触：

- GitHub。
- 官方文档。
- 包管理器。
- Docker 镜像。
- npm、pip、apt、Homebrew 等软件源。
- 在线评测系统。
- 远程服务器。

其中很多服务在国内网络下并不稳定。你需要逐渐理解几个概念：

- DNS：负责把域名解析成 IP。
- 代理：让部分网络请求走指定通道。
- VPN：通常是更完整的网络隧道。
- 镜像源：把国外软件源同步到国内，提升下载速度。
- 防火墙：系统、校园网、路由器或服务器都可能拦截连接。

相关内容较为敏感，不适合公开展开，但对计算机相关专业同学非常重要。遇到 GitHub 打不开、包下载失败、`Connection timed out`、`Could not resolve host` 之类问题时，不要立刻怀疑自己代码写错了，先判断是不是网络问题。

## 数据备份

电脑坏了可以买新的，数据丢了不一定能找回来。

一定要备份：

- 课程作业。
- 实验报告。
- 项目代码。
- 简历。
- 证件扫描件。
- 重要聊天文件。
- 毕业设计、论文、竞赛材料。

最简单的备份策略：

- 代码：用 Git，并推送到 GitHub/Gitee/学校 GitLab。
- 文档：同步到可靠网盘或云盘。
- 大文件：定期复制到移动硬盘。
- Mac：可以使用 Time Machine。
- 重要账号：开启双重验证，保存好恢复码。

更正式的说法叫 3-2-1 备份：至少 3 份数据，存放在 2 种不同介质，其中 1 份放在异地。你不一定一开始做到这么标准，但至少不要让重要文件只存在一个硬盘里。

还有一个常见误区：Git 不等于完整备份。Git 很适合管理代码历史，但你的数据库、模型文件、环境配置、大型素材、未提交文件，可能根本没进 Git。
